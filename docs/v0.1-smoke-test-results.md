## Implementation invariants verified on-device

- **LADDER-only autoregulate** — picker filter (Hip Thrust + Pull-up excluded) AND detail-screen button gate (Hip Thrust button disabled with explanatory label).
- **Signature flags surfaced on detail** — `status`, `load_equipment_id`, `band_eligible` all rendered with non-null values; nullable flags (`objective_override`, `assist_subtype`, `assist_unit`) correctly omitted.
- **Dynamic tier max from ladder** — Back Squat (3-rung) shows (0..2); empty selection shows (0..0); stepper bounds enforced.
- **Pre-fill auto-load** — selecting a movement defaults `current_load` to that movement's `load_floor`.
- **Delta ternary** — ON_TARGET → "unchanged"; TOO_EASY → "+10.0 lb" (positive prefix).
- **Typed error mapping** — server down → red health dot + `ApiError.Network.humanMessage()` ("Network error — is the server reachable?") + Retry; no crash, no silent failure.
- **In-place recovery** — Retry triggers ViewModel reload; UI state transitions Loading → Success without app restart.

## Issues found

- `tmux` not installed on myflix — server started via `nohup` instead. Survives SSH disconnection but does not survive a server reboot. Follow-up: create a systemd user service.
- Port 8000 was not open in myflix firewall — opened permanently via `sudo firewall-cmd --add-port=8000/tcp --permanent && firewall-cmd --reload`.
- Cosmetic deferrables already logged in `.superpowers/sdd/progress.md` for v0.1.x cleanup (delta double-formatting, HorizontalDivider double-border, factored ErrorRetryBox).
